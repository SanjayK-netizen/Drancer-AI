Drancer
Drancer is a personal AI voice assistant built by Sanjay K. that runs entirely on your own computer, with no cloud services, no API keys, and no internet connection required once everything is set up. The whole point of building it this way, rather than using an existing assistant like Siri or Alexa, was privacy — your voice, your conversations, and your data never leave your machine.
The way it works is simple in concept. When you speak, Drancer records your voice through the microphone and converts it into text using Whisper, a speech recognition model from OpenAI that runs locally rather than through an API. That text is then handed off to phi3:mini, a small but capable AI language model running through Ollama on your own hardware, which generates a reply. Finally, that reply is converted back into spoken audio using pyttsx3, a text-to-speech engine, so Drancer talks back to you the same way you talked to it.
To get it running, you first need Ollama installed on your system along with the phi3:mini model pulled down, which you do by running ollama pull phi3:mini in your terminal. From there, you install the Python dependencies with pip install openai-whisper numpy sounddevice pyttsx3. Depending on your operating system, you'll also need a couple of system-level tools — on Windows that means installing ffmpeg and making sure it's on your PATH, on Mac it's brew install ffmpeg portaudio, and on Linux it's sudo apt install ffmpeg portaudio19-dev. Once all of that is in place, running python voice_loop.py starts the assistant, and you simply speak when you see "Listening..." appear in the terminal, ending the session at any time with Ctrl+C.
The project itself is split into two files that each handle a distinct job. brain.py is the thinking layer — it holds the logic that talks to phi3:mini through Ollama and can be tested entirely on its own by running python brain.py, without any voice component involved at all. voice_loop.py is the layer built on top of that, responsible for listening to your microphone, transcribing what you said, and speaking the reply back out loud.
As it stands, both major phases are complete: the text-based brain from Phase 1, and the voice input and output layer from Phase 2. That said, there are a few known limitations worth being upfront about. Right now the assistant records in a fixed five-second window rather than detecting when you naturally stop talking, which means short sentences can leave dead air and longer ones can get cut off. There's also no wake word yet, so the script has to be manually started each time rather than listening passively in the background. And because the system runs entirely on CPU — since AMD graphics cards don't support the CUDA acceleration Whisper normally relies on — there's a short but noticeable delay between finishing a sentence and hearing Drancer's reply.
